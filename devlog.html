DISASTER LOG

Every build breaks before it works. This is where I document the breaks — what failed, how long I was stuck, and what changed after. No cleanup. No retroactive wisdom. Just what actually happened.

BITS & BUILDS — THE SITE NOBODY COULD NAVIGATE RESOLVED
what broke

I shipped my portfolio and immediately got the same note from multiple reviewers: the navbar buttons didn't work. Not one reviewer. Multiple. The About, Projects, and Goals links in the nav were broken — users couldn't actually get anywhere. I'd been testing it myself the whole time and somehow never caught it.

how long i was stuck

I wasn't stuck on the bug itself — once I knew it existed, the fix was straightforward. What stung was that I shipped something I thought was done, and it wasn't. The real cost was the gap between what I assumed was working and what was actually working for someone else opening it fresh.

"The topbar isn't fully working, the music button isn't working either, when I click a project, I don't get redirected." — Voter #8
what changed

I learned that testing your own work is almost useless because you already know where to click. The only real test is handing it to someone who has never seen it. Now before I ship anything, I ask at least one person to open it cold and tell me where they get stuck. I also learned that public feedback — even when it's blunt — is the fastest way to find what you missed.

EGG TIMER — WHEN AI WRITES YOUR CODE AND YOU DON'T UNDERSTAND IT RESOLVED
what broke

I wanted pixel art animations in the egg timer — little eggs bobbing and flickering based on the timer stage. Canvas animation was beyond what I'd covered, so I asked AI to generate the drawing code. It worked. The eggs moved. But I had no idea what any of it was actually doing, and it didn't match the rest of my codebase at all. I'd technically shipped a feature I couldn't explain.

how long i was stuck

The animation ran fine, which meant there was no obvious failure to fix. The problem was quieter: I had working code I didn't own. I sat with that discomfort for a while before deciding to do something about it.

I went through the code properly and made sure I understood what each part was doing before I let it stay in my project.
what changed

I rewrote the entire animation in my own style after learning how requestAnimationFrame creates a smooth loop, how Math.sin generates the bobbing motion, and how clearRect wipes the canvas each frame. Then I rewrote it from scratch to match my own code. The final version does exactly the same thing as the AI-generated one — but now I can explain every line. Using AI to understand something is fine. Using AI to ship something you can't explain is a different thing entirely, and I won't do that again.

SATSCOPE — BUILDING A 3D SATELLITE IN THREE.JS (IN PROGRESS) IN PROGRESS — 59%
what i'm building

An interactive 3D satellite explorer using Three.js — a full satellite built from code and geometry, orbiting a textured Earth with 6,000 scattered background stars. Each component (solar panels, antenna dish, thrusters, payload, star sensor) is clickable and shows real information about what it does.

bugs so far

Mostly typos and wrong variable names in the geometry setup. The antenna dish shape is built using LatheGeometry — you give it a list of points tracing a curve, and Three.js spins it into a 3D bowl automatically. Getting the curve coordinates right took a lot of trial-and-error on position and size values. I'm getting faster at spotting these, but they still eat time.

Instead of writing the same thruster code four times I put it in one function and called it four times with different positions. If I want to change anything, I only change it in one place.
what i'm learning

Three.js forces you to think in 3D space — every object has a position, rotation, and scale on three axes, and getting things to sit correctly relative to each other is genuinely hard. The biggest lesson so far is about functions: repetition in code is a warning sign. When I found myself writing the same thruster geometry four times, I stopped and refactored. The instinct to clean up before moving forward is something I'm actively building.
